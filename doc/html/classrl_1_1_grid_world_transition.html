Detailed Description

Simple transition function which is known a priori for the grid world and does not have to be learned.

It would also be possible to use TransitionStlMap and pre-fill it with all known transitions, but this would be overshoot for this simple problem.

Parameters
_maxXand _maxY: dimensions of the grid world
_goalXand _goalY: coordinates of goal (index starts with 0 and can be maximum _maxX! and _maxY-1 respectively)
_blockXand _blockY: coordinates of block/wall (index starts with 0 and can be maximum _maxX! and _maxY-1 respectively)
_sideActionProbabilityprobability that action to be perform fails [0..1]. With this probability, other available side actions are executed, if they are possible. If no side action is possible, the main action is performed with possibility 1. Example: if _sideActionProbability=0.1, and Action is "UP", and both "LEFT" and "RIGHT" are accessible states, "UP" is going to be performed with probability 0.8, and "LEFT" and "RIGHT" with probabilities 0.1 repsectively.
